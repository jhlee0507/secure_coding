# Security Policy

## 지원 범위

현재 기본 브랜치의 최신 버전만 보안 수정 대상입니다. 이 프로젝트는 교육용이며 실제 금융 처리나 개인정보 저장을 목적으로 하지 않습니다.

## 취약점 제보

공개 Issue에 공격 절차, 비밀번호, 세션, 개인정보를 게시하지 마세요. 저장소 소유자에게 비공개 채널로 다음 내용을 전달합니다.

- 영향을 받는 기능과 버전 또는 커밋
- 재현에 필요한 최소 단계
- 예상 결과와 실제 결과
- 가능한 경우 완화 방법

제보를 받은 뒤 재현 여부와 심각도를 확인하고, 수정·테스트·배포 순서로 처리합니다. 실제 데이터가 노출된 경우 자격 증명 폐기와 사고 대응을 우선합니다.

## 운영 전 확인사항

- `APP_ENV=production`, 무작위 `SECRET_KEY`, HTTPS, `COOKIE_SECURE=true`
- 디버그 모드 비활성화
- Redis 등 공유 rate-limit 저장소
- 데이터베이스 파일의 최소 권한과 암호화된 백업
- 리버스 프록시의 요청 크기·시간 제한
- 의존성 취약점 점검 및 정기 업데이트
- 감사 로그의 접근 제한과 보존 기간 설정
